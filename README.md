# 📊 Ana-bro
### Enterprise E-Commerce Analytics Dashboard

A high-performance, multi-page analytics dashboard built with Python, Polars, and Dash — styled with a dark-mode glassmorphism UI. Analyses 2.6M+ e-commerce events from the [2019 Kaggle E-Commerce dataset](https://www.kaggle.com/datasets/mkechinov/ecommerce-behavior-data-from-multi-category-store).

---

## ✨ Features

- **Overview** — Revenue KPIs, daily sales time-series, and top 10 brands by revenue
- **Conversion Funnel** — Interactive view → cart → purchase funnel with live category filtering and drop-off rates
- **Temporal Patterns** — Day-of-week × hour-of-day heatmap revealing when users browse and buy
- **Statistical Deep Dive** — Violin plots and scatter plot of price vs sales volume per product
- **Distributions** — Dynamic Pie charts, Box plots, and Histograms with switchable dropdown metrics
- **Advanced** — 3D Scatter plot, Animated Bar Race, and Animated Time Series — all with Play/Pause controls

## 🚀 Getting Started

### 1. Clone the repo
```bash
git clone https://github.com/your-username/ana-bro.git
cd ana-bro
```

### 2. Get the dataset
Download **2019-Oct.csv** from [Kaggle](https://www.kaggle.com/datasets/mkechinov/ecommerce-behavior-data-from-multi-category-store) and place it at:
```
data/raw_data.csv
```

### 3. Create a virtual environment

**Mac / Linux**
```bash
python -m venv venv
source venv/bin/activate
```

**Windows (PowerShell)**
```powershell
python -m venv venv
venv\Scripts\activate
```

### 4. Install dependencies
```bash
pip install -r requirements.txt
```

### 5. Run the data pipeline
```bash
python data_pipeline.py
```
This scans the raw CSV using Polars LazyFrames, engineers features, and streams the result to `data/optimized_data.parquet` — without loading the full file into RAM.

Expected output:
```
🔍 Scanning CSV with Polars LazyFrame...
⚙️  Processing and writing to Parquet...
✅ Done! Optimized file saved → data/optimized_data.parquet
   Rows    : 2,600,000+
   Event types : ['view', 'cart', 'purchase']
   Date range  : 2019-10-01 → 2019-10-31
```

### 6. Launch the dashboard

> ⚠️ This project uses **Waitress** instead of Werkzeug's dev server to avoid `MemoryError` on large datasets.

**Windows (PowerShell) — recommended:**
```powershell
waitress-serve --host=127.0.0.1 --port=8050 --threads=16 --connection-limit=1000 --channel-timeout=60 app:server
```

**Mac / Linux:**
```bash
waitress-serve --host=127.0.0.1 --port=8050 --threads=8 app:server
```

**Or use the dev server for quick testing:**
```bash
python app.py
```

Open **http://127.0.0.1:8050** in your browser.

> 💡 **Tuning threads:** Match `--threads` to your CPU thread count. An i5-12th gen benefits from `--threads=16`.

## 📸 Pages

### 📊 Overview
KPI cards for total revenue, orders, and unique users. Daily revenue time-series with area fill and top 10 brands ranked by revenue.

### 🔻 Conversion Funnel
Select any product category from the dropdown to see how users move through the view → cart → purchase pipeline. Conversion rates and drop-off percentages update live.

### 🌍 Temporal Patterns
A 7×24 heatmap (day of week × hour of day) showing activity intensity. Toggle between views, cart adds, and purchases. A secondary bar chart shows purchases broken down by hour.

### 📈 Statistical Deep Dive
Violin plots comparing price distributions across the top 5 categories. A scatter plot maps every product's average price against its total sales volume — making high-ticket vs high-volume products immediately visible. Pre-built at server startup for instant loads.

### 🥧 Distributions *(new)*
Three fully dynamic charts — each with its own dropdown:
- **Pie Chart** — switch between category share, brand revenue, event type split, or hour of day
- **Box Plot** — compare price distributions by top categories, brands, event type, or day of week. Uses 2000-point random sampling per group for accuracy without browser slowdown
- **Histogram** — view price frequency distribution for purchases, views, or cart events. Pre-binned server-side for instant rendering

### 🚀 Advanced *(new)*
Three animated, interactive charts — each with Play/Pause button and date slider:
- **3D Scatter Plot** — explore price × hour × sales volume relationships across top categories
- **Animated Bar Race** — watch top 10 brands race by revenue, order count, or avg price across every day of October
- **Animated Time Series** — see daily revenue, orders, or unique users build up day by day through the month

## ⚙️ Data Pipeline Details

The pipeline uses **Polars LazyFrames** (`pl.scan_csv`) to process the multi-gigabyte CSV without RAM overload:

1. **Lazy scan** — reads schema only, no data loaded into RAM
2. **Column selection** — keeps only the 7 required columns
3. **Null filtering** — drops rows missing price, category, or brand
4. **Feature engineering** — parses datetime, extracts `date`, `hour`, and `category_top`
5. **sink_parquet** — streams directly to a compressed Snappy Parquet file

## 🧠 Performance Optimisations

| Optimisation | Details |
|---|---|
| **Global parquet load** | Data loaded once at startup in `app.py`, shared across all pages via `get_df()` |
| **Pre-built Statistics figures** | Violin + scatter figures built at startup, serialized to JSON via `dcc.Store`, never recomputed |
| **Box plot sampling** | 2000-point random sample per group — statistically accurate (±2.2% margin) with tiny browser payload |
| **Histogram pre-binning** | Polars bins data into 60 buckets server-side, sends bar heights not raw points |
| **Waitress WSGI server** | Replaces Werkzeug dev server, handles large payloads without MemoryError |

## ⚠️ Known Issues & Fixes

**`MemoryError` on startup**
Werkzeug's dev server tries to read 10MB request chunks at once, crashing on large datasets. Fixed by switching to Waitress and adding a `MAX_CONTENT_LENGTH = 32MB` cap in `app.py`.

**Waitress queue depth warnings**
```
WARNING:waitress.queue:Task queue depth is 2
```
Not an error — requests briefly queue while workers process callbacks. Increase `--threads` to reduce these.

**Dropdown menu hidden behind charts**
Dash dropdowns get clipped by parent containers. Fixed with `overflow: visible` and `z-index: 9999` on the dropdown wrapper div.

**Box plot all stacking at x=0**
`go.Box` with pre-aggregated stats (`q1=`, `median=`) stacks all traces at position 0 — it doesn't support side-by-side layout in that mode. Fixed by switching to `y=` with 2000-point sampled raw data per group.
